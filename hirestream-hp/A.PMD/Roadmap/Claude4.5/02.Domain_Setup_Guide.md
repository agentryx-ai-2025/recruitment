# Domain Setup Guide: hirestream.osipl.dev

## Overview
This guide will help you bind your HireStream application running on port 5000 to the domain **hirestream.osipl.dev**.

---

## Prerequisites

✅ Application running on `localhost:5000`  
✅ Domain `hirestream.osipl.dev` registered  
⚠️ Nginx installed (will install if needed)  
⚠️ DNS A record pointing to your server IP  

---

## Step 1: Verify DNS Configuration

First, ensure your domain's DNS is properly configured:

```bash
# Check DNS propagation
nslookup hirestream.osipl.dev

# Or use dig
dig hirestream.osipl.dev
```

**Required DNS Setup:**
- Add an **A record** for `hirestream.osipl.dev` pointing to your server's public IP address
- Wait for DNS propagation (can take 5 minutes to 24 hours)

---

## Step 2: Install Nginx (if not installed)

```bash
# Check if Nginx is installed
nginx -v

# If not installed (Ubuntu/Debian):
sudo apt update
sudo apt install nginx -y

# Start and enable Nginx
sudo systemctl start nginx
sudo systemctl enable nginx

# Check status
sudo systemctl status nginx
```

---

## Step 3: Deploy Nginx Configuration

```bash
# Copy the configuration file
sudo cp /home/subhash.thakur.india/Projects/HireStream/Roadmap/Claude4.5/nginx-hirestream.conf /etc/nginx/sites-available/hirestream

# Create symbolic link to enable the site
sudo ln -s /etc/nginx/sites-available/hirestream /etc/nginx/sites-enabled/

# Test Nginx configuration
sudo nginx -t

# If test passes, reload Nginx
sudo systemctl reload nginx
```

---

## Step 4: Configure Firewall

```bash
# Allow HTTP and HTTPS through firewall
sudo ufw allow 'Nginx Full'

# Or manually:
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# Check firewall status
sudo ufw status
```

---

## Step 5: Keep Application Running with PM2

Instead of running `npm run dev` manually, use PM2 for production:

```bash
# Install PM2 globally (if not installed)
sudo npm install -g pm2

# Start the application with PM2
cd /home/subhash.thakur.india/Projects/HireStream
pm2 start npm --name "hirestream" -- run dev

# Save PM2 configuration
pm2 save

# Set PM2 to start on boot
pm2 startup

# Check status
pm2 status
pm2 logs hirestream
```

---

## Step 6: Test the Setup

```bash
# Test locally first
curl -I http://localhost:5000

# Test via domain (once DNS is propagated)
curl -I http://hirestream.osipl.dev

# Test in browser
# Open: http://hirestream.osipl.dev
```

---

## Step 7: SSL Certificate (HTTPS) - Recommended

For production, you should enable HTTPS using Let's Encrypt:

```bash
# Install Certbot
sudo apt install certbot python3-certbot-nginx -y

# Obtain SSL certificate
sudo certbot --nginx -d hirestream.osipl.dev

# Certbot will automatically:
# 1. Obtain the certificate
# 2. Modify your Nginx config
# 3. Set up auto-renewal

# Test auto-renewal
sudo certbot renew --dry-run
```

After SSL setup, your site will be accessible at:
- ✅ `https://hirestream.osipl.dev` (secure)
- ⚠️ `http://hirestream.osipl.dev` (will redirect to HTTPS)

## Troubleshooting

### Issue: "502 Bad Gateway"
**Cause**: Application not running on port 5000  
**Fix**: 
```bash
pm2 status
pm2 restart hirestream
```

### Issue: "Connection refused"
**Cause**: Firewall blocking port  
**Fix**:
```bash
sudo ufw allow 80
sudo ufw allow 443
```

### Issue: Domain not resolving
**Cause**: DNS not propagated yet  
**Fix**: Wait 1-24 hours, check with `nslookup hirestream.osipl.dev`

### Issue: Nginx won't start
**Cause**: Configuration error  
**Fix**:
```bash
sudo nginx -t  # Check for errors
sudo tail -f /var/log/nginx/error.log  # View error logs
```
