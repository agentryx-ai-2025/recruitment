# HireStream — v1.5.1 Release Notes

**Release date:** 2026-04-15 (same-day patch on top of v1.5.0)
**Theme:** *Candidate-side uplift — self-service compliance, portfolio PDF, calendar export, welfare reply, references*

Same-day follow-up release that brings the candidate portal to parity with the agent overhaul. 7 new candidate-facing capabilities + 1 new backend router.

## New for the candidate

1. **Pre-Departure Compliance self-service panel** — Documents tab → new section lets the candidate fill passport, ECR, PCC, medical, IELTS, PDO, PBBY details themselves. Saves instantly.
2. **Portfolio PDF export** — Documents tab → "My Portfolio PDF" card → one-click branded PDF download.
3. **"Add to calendar (.ics)" on interviews** — Upcoming Interviews card → per-interview link next to the details.
4. **Offer letter PDF button** — appears on the green Offer banner in Application detail → "Download offer letter (PDF)".
5. **Welfare reply card** — auto-surfaces on Overview when candidate has an accepted/active placement → one-line "How are you doing?" send-update box.
6. **Interview prep tips card** — auto-surfaces on Overview when upcoming interviews exist → 5 practical tips.
7. **Professional References** — Documents tab → "Professional References" section. Candidate adds up to 3 references. Agents see them during background checks.

## New API endpoints (all under `/api/v1/me`)

| Method | Path | Purpose |
|---|---|---|
| PATCH | `/api/v1/me/compliance` | Candidate self-edit compliance fields |
| GET   | `/api/v1/me/references` | List own references |
| POST  | `/api/v1/me/references` | Add reference |
| DELETE | `/api/v1/me/references/:id` | Remove reference |
| POST  | `/api/v1/me/placements/:id/welfare-reply` | Candidate welfare update |
| GET   | `/api/v1/me/profile.pdf` | Portfolio PDF download |
| GET   | `/api/v1/me/interviews/:id.ics` | Calendar export |
| GET   | `/api/v1/me/placements/:id/offer-letter.pdf` | Offer letter PDF (redirects to agent endpoint with same auth) |

## Schema changes

- New table `candidate_references` (id, candidate_id, name, relationship, email, phone, organization, contacted, contacted_at, notes, created_at)
- `placements` gains `candidate_welfare_note`, `candidate_welfare_note_at`

## Backward compatibility

All additive. No breaking changes. No existing endpoints modified.
