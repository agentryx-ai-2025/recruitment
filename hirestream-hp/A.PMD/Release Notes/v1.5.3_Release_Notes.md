# HireStream — v1.5.3 Release Notes

**Release date:** 2026-04-15
**Theme:** *Employer role — proper workflow overhaul (decision-maker, not recruiter)*

v1.5.2 gave employers access to agent pages. v1.5.3 rebuilds the employer experience around the **real overseas placement workflow**: employer is a decision-maker reviewing the agency's curated shortlist — not a front-line recruiter.

## The workflow the portal now reflects

```
Candidate applies
    ↓
Agency screens, reviews, shortlists      ← agency's 90% of work
    ↓
EMPLOYER: "Approve for interview"
    ↓  OR "Request replacement candidates"
Interview (agency schedules, employer attends)
    ↓
EMPLOYER: "Select" or "Reject"
    ↓
EMPLOYER: Issue appointment letter
    ↓
Agency handles visa / PDO / PBBY / travel
    ↓
Post-placement welfare — both agency AND employer contribute
```

## What's new

### 1. New page: `/employer/review/:jobId` (the decision queue)
Purpose-built employer-only UI. Does **not** reuse the agent's applicant pipeline page.

- **Purple/indigo brand** (vs agent's blue) — visually distinct role
- **Hero requisition card** — target hires, deadline, priority (standard/urgent/critical), progress bar
- **Review queue** — shows only `shortlisted` / `interview_scheduled` / `selected` (filters out agency's noise)
- **Stage relabelling** — "Shortlisted" shows as "Awaiting your review"
- **Decision actions per candidate**:
  - **Approve for interview** — agency is notified to schedule
  - **Request replacement** — agency is notified with the reason
  - **Select this candidate** (after interview)
  - **Reject** with feedback
- **Side-by-side comparison** — select 2+ candidates → Compare modal with skills, experience, match score, decision status in one view
- **Request more candidates** — dedicated modal sends a formal request to the agency for additional candidates with count + reason

### 2. Redesigned employer Overview

- **Hero: "Awaiting your decision"** — huge number + 3 preview cards to click into. This is the employer's primary queue and it's front-and-centre.
- **Active Requisitions panel** — each requisition shows:
  - Title + priority badge (🔥 Critical / ⚡ Urgent)
  - Target hires + progress bar (filled / target)
  - Awaiting review count in purple
  - Approved + selected counts
  - Deadline if set
  - Click → review queue for that requisition

### 3. Employer welfare contribution
Employer can now **add their own welfare note** on each placement ("employee performing well / had issues"), alongside the agency's 30/60/90 check-ins and the candidate's proactive notes.

### 4. Requisition fields (schema)
Jobs now carry:
- `targetHires` — how many positions to fill
- `hiringDeadline` — target date
- `priority` — standard | urgent | critical
- `employerNotes` — internal notes

### 5. Application decision tracking (schema)
Each application carries an **employer decision** independent of the pipeline status:
- `approved_for_interview`
- `replacement_requested`
- `selected_by_employer`
- `rejected_by_employer`

This preserves the employer's audit trail even if the agency subsequently changes pipeline status.

## New API endpoints

All under `/api/v1/employer/*`:

| Method | Path | Purpose |
|---|---|---|
| GET | `/requisitions` | List employer's requisitions + candidate counts |
| GET | `/review-queue` | Cross-job queue of candidates awaiting decision |
| POST | `/applications/:id/approve-for-interview` | Approve + notify agency |
| POST | `/applications/:id/request-replacement` | Reject + request replacement |
| POST | `/requisitions/:jobId/request-more` | Ask agency for more candidates |
| PATCH | `/requisitions/:id` | Update requisition fields |
| PATCH | `/placements/:id/welfare-note` | Employer welfare note |

## Why this matters

Before v1.5.3, the employer experience was a thin pass-through of the agent's UI. It worked technically but misrepresented the real workflow: overseas employers *don't* sift raw applications — that's what they pay the agency to do.

Now the employer's portal reflects how NHS, Aramco, Siemens, Marriott, etc. actually operate: they receive a curated shortlist, approve/reject with reasons, interview, select, and issue appointment letters. The agency remains the operational engine.
