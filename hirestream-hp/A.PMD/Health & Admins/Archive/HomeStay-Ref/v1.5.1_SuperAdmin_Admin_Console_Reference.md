# v1.5.1 — Super Admin & Admin Console Reference

**Document Date:** April 2026  
**Applies to:** v1.5.0 codebase (adventure.osipl.dev / hptourism-adv)  
**Purpose:** Complete reference of every page, feature, and API capability available to `super_admin` and `admin` roles — who can do what, and where.

---

## Role Hierarchy (Admin Tier)

| Role | Access Level | Notes |
|------|-------------|-------|
| `super_admin` | Full — every admin feature | Manages the system itself |
| `admin` | Operational — most features, no destructive ops | Day-to-day portal management |
| `system_admin` | Technical — monitoring + tools, no business data edits | Dev/ops only |
| `admin_rc` | Narrowly scoped — RC legacy queue only | Not covered in detail here |

## 2. Ops Console (`/admin/ops`) — Tab-by-Tab

The Ops Console is the central health and monitoring dashboard. Auto-refreshes at configurable interval (15s / 30s / 1m / Off).

### Overview Tab
- **Health Score** — 0–100 (Healthy / Degraded / Unhealthy / Critical), computed from active alerts
- **Smart Alerts Banner** — up to 15 alert conditions, each with snooze options (1h / 4h / 12h / 24h / 7d)
- **Process Status** — ONLINE/STOPPED/ERRORED, uptime, restart count, PID, Node version
- **Node Memory** — Heap used/total, RSS, external memory
- **External Dependencies** — DB ping + latency, DB pool (total/idle/waiting), HimKosh gateway, Session Store, Upload Storage, SMTP configured (yes/no), SMS configured (yes/no)
- **Active Sessions** — count by role, list of currently online officers
- **Environment Variables** — presence check only (values never shown)

### Resources Tab
- **CPU** — current %, core count, model, load averages (1m / 5m / 15m), rolling sparkline
- **Memory** — system total/free/used %, Node heap/RSS/external, rolling sparkline
- **Disk Storage** — root partition %, uploads dir size, backups dir size (amber >75%, red >90%)
- **Node.js Runtime** — event loop delay ms, active handles, active requests, PID, platform, architecture

### Pipeline Tab (Application Health)
- **App Count Grid** — all services × all statuses (Draft / Submitted / Under Scrutiny / With DTDO / Payment Pending / Approved / Rejected)  
  Services: Homestay, Tourism Unit, Restaurant, Travel Agent, Guide, Photographer
- **Stuck Apps (auto-detected):**
  - Payment pending > 48h (name, mobile, last update)
  - Submitted, no DA assigned > 5 days
  - Under scrutiny > 10 days
  - Old drafts > 7 days
- **Today's Activity** — all status changes since midnight
- **Payment Health** — counts by status, oldest pending age, pending total (₹), monthly/yearly collected
- **Certificate Expiry Watchlist** — expiring in 7 days (urgent list), expiring in 30 days (count), already expired count

### Backups Tab
- Scheduler status (running/stopped, cron schedule, next run)
- Last backup — status, timestamp, error detail if failed
- Backup history — last 15 backups with size, status, age
- Link to full Backup Management page

### Signals Tab
- Period filter: Today / 7 days / 30 days / Custom date range
- Application submission trends, approval rates, payment success rates over selected period

### Logs Tab
- Last 2MB of Pino JSON log file
- Filters: log level (info/warn/error/fatal), module, status code, application ID, free-text search
- View modes: Grouped / Timeline
- Expandable stack traces for error entries

### Log Query Panel (within Logs Tab)
- RegExp pattern search across logs (`-i` case-insensitive, `-v` invert match flags)
- 6 quick-example patterns built in
- Raw + parsed output; expandable JSON results

### Lookup Tab
- Search by applicant name, mobile number, or application number
- Works across homestay + tourism unit + all service applications
- Side panel shows full application detail + payment history

### DB Explorer Tab (`super_admin` / `system_admin` only for custom SQL)
- **7 Pre-built reports:**
  - By District
  - By Category
  - Revenue Summary
  - Stuck Drafts
  - Officer Activity
  - Cert Expiry (30 days)
  - Payment Reconciliation
- **SQL Sandbox** — SELECT-only, 5-second statement timeout, max 500 rows, always rolled back

### System Tab
- TCP connection breakdown (ESTABLISHED / TIME_WAIT / LISTEN / etc.)
- File descriptor gauge — current open / system limit
- Zombie process detector
- Listening ports list
- Top processes by CPU/memory
- **Process Actions** (super_admin only):
  - Restart application (2-step confirm)
  - Clear logs / flush PM2 log files (2-step confirm)

## 3. Super Console (`/admin/super-console`) — Section-by-Section

Super_admin only. Full system control panel.

| Section | Capabilities |
|---------|-------------|
| **System Overview** | DB connection status, pool stats, version info |
| **Reset Zone** | Granular data reset (by table/type), full DB wipe (with confirmation), selective seed |
| **Payment Settings** | Test mode toggle (uses simulated gateway), payment workflow mode (submit-first vs. pay-first), payment gateway config (Kotak credentials) |
| **Communications** | Email (SMTP) config — host, port, user, from address; test-send button. SMS config — provider, API key; test-send button |
| **Notifications** | Portal-wide notification preferences |
| **Security** | Rate limit configuration (per-endpoint overrides), CAPTCHA toggle |
| **District Staff** | Seed DA accounts, seed DTDO accounts, seed system accounts by district |
| **Smoke Tests** | Run pre-built end-to-end checks (auth, DB, HimKosh, storage) from the browser |
| **Test Data** | Create synthetic test applications for staging/demo |
| **Inspection Config** | Per-service inspection requirement toggle (mandatory / no_need) |

## 4. User Management (`/admin/users`)

Available to `admin`, `super_admin`, `system_admin`.

- **List all users** — searchable, filterable by role and district
- **Active Sessions panel** — who is currently logged in
- **Create user** — any role, any district
- **Edit user** — name, mobile, district, role
- **Activate / Deactivate** — block a user without deleting their data

### Role assignment capabilities

| Acting Role | Can create/edit |
|-------------|----------------|
| `super_admin` | Any role including other super_admins |
| `admin` | `dealing_assistant`, `district_tourism_officer`, `inspector`, `state_officer`, `payment_officer`, `property_owner` |
| `system_admin` | View only (no writes) |

## 5. Settings (`/admin/settings`)

### Available to `admin` + `super_admin` + `system_admin` (read/toggle)

| Setting | Description |
|---------|-------------|

| **CAPTCHA** | Enable/disable OTP captcha on login |

| **Inspection Config** | Set inspection requirement per service type per application kind |
| **Payment Gateway** | View/update Kotak gateway credentials |
| **Rate Limits** | View current rate limit thresholds |
| **Max Reverts** | View max application revert limit |
| **Renewal Window** | View renewal eligibility window (days before expiry) |

### `super_admin` only (write/toggle)

| Setting | Description |
|---------|-------------|
| **Payment Workflow** | Toggle pay-first vs. submit-first workflow |
| **Payment Submit Mode** | Configure submit-mode behavior |
| **Rate Limits** (write) | Change per-endpoint rate limit values |
| **Max Reverts** (write) | Change max revert count |
| **Inspection Backdate Days** | Allow backdating of inspection reports by N days |
| **Renewal Window** (write) | Change renewal window duration |
| **Super Console Toggle** | Enable/disable Super Console section visibility |

## 6. Backup Management (`/admin/backup`) — `super_admin` only

- **Run backup now** — triggers `pg_dump --format=custom --compress=9`
- **Backup scheduler** — start/stop cron schedule, set cron expression
- **Backup history** — list with size, timestamp, status
- **Download** — download any backup file (binary or SQL format)
- **Delete** — remove old backup files
- **Scheduler status** — running/stopped, next scheduled run

## 7. Reports & Analytics

### Reports & Insights (`/admin/payment-reports`)

Available to `admin`, `super_admin`, `system_admin`, `state_officer`, `district_tourism_officer`, `payment_officer`.

| Report | Description |
|--------|-------------|
| Collections | Total fees collected by period and district |
| Payments | Individual payment records with gateway reference |
| Refundable | Applications where a refund may be due |
| Summary | High-level dashboard numbers |
| DDO Codes | Payments grouped by DDO/Treasury code |
| Operations | Officer-level activity (applications processed per officer) |
| Applications | Application counts by service, status, district |
| District Performance | DA/DTDO throughput and SLA metrics |
| Existing RC | Onboarding status of legacy RC holders |
| Overpaid | Applications where amount collected exceeds the fee schedule |

### Treasury Forecast (`/admin/treasury-forecast`)

Available to `state_officer`, `super_admin`, `admin`, `system_admin`.

- Forward projection of fee revenue by service type and district
- Based on current pipeline + historical approval rates

### Reconciliation Engine (`/admin/reconciliation`)

Available to `admin`, `super_admin`, `system_admin`, `payment_officer`.

- Match HimKosh gateway transaction records against application payment records
- Surface orphaned payments (gateway success, no DB record) and pending mismatches
- Manual reconciliation actions: mark as reconciled, flag for review

## 8. RC Legacy Queue (`/admin/rc-applications`)

Available to `admin_rc`, `admin`, `super_admin`, `system_admin`.

- List of all legacy RC onboarding applications (owners who registered under old paper system)
- Detail view with full application data

## 9. LGD Import (`/admin/lgd-import`)

Available to `admin`, `super_admin`, `system_admin`.

- Import the Lok Granth Data hierarchy (State → District → Block → GP → Village)
- Used to populate district/tehsil dropdowns across all service forms
- Import is idempotent — re-running updates existing records

## 11. Smart Alerts — 15 Conditions

The Ops Console evaluates these continuously. Each has a recommended action and can be snoozed.

| # | Alert | Trigger | Severity |
|---|-------|---------|----------|
| 1 | DB Unreachable | `SELECT 1` fails | Critical |
| 2 | Memory Critical | Node heap > 400 MB | Critical |
| 3 | Memory Elevated | Node heap > 280 MB | Warning |
| 4 | Memory Leak Trend | Heap increasing monotonically over last 6 samples (3 min) | Warning |
| 5 | Event Loop Blocked | Event loop delay > 100 ms | Critical |
| 6 | Event Loop Slow | Event loop delay > 50 ms | Warning |
| 7 | Disk Critical | Root partition > 90% | Critical |
| 8 | Disk Low | Root partition > 75% | Warning |
| 9 | Stuck Payments | Any payment_pending application > 48h | Warning |
| 10 | Payment/Status Mismatch | Gateway shows success but application still payment_pending | Warning |
| 11 | No Recent Backup | No backup in > 25h | Warning |
| 12 | Backup Failed | Last backup status = failed | Warning |
| 13 | Certs Expiring Soon | Any RC cert expiry within 7 days | Info |
| 14 | Process Crash-Looping | > 3 restarts in < 1h | Critical |
| 15 | File Descriptors Critical | FD usage > 90% of system limit | Critical |
