# v1.5.2 — Testing Module (HireStream)

**Document Date:** April 2026  
**Project:** HireStream — Overseas Placement Portal (HPSEDC)  
**Purpose:** Comprehensive automated testing framework — every feature gets a test during development, not after.  
**Status:** PLANNED — Ready for Implementation  
**New Module — Not present in HP Tourism (fills a gap)**

---

## Why This Module Exists

The HP Tourism project shipped without a formal testing module. The result: manual verification for every release, regression bugs caught in production, and troubleshooting sessions that could have been prevented by a 30-second test run.

HireStream fixes this from the start. **The rule is simple: no feature ships without its test.**

**Testing philosophy:**
- Tests are written **during** development, not bolted on after
- Every API endpoint gets an integration test before the PR merges
- Every UI component gets a component test for its critical paths
- Every multi-step workflow gets an E2E test
- The test suite runs automatically on every build
- A human test plan is generated from the automated tests — manual testing becomes verification, not discovery

---

## Testing Stack

| Layer | Tool | Purpose |
|-------|------|---------|
| Unit Tests | Jest | Pure functions, utilities, validators, schema logic |
| API Integration Tests | Jest + Supertest | Every `/api/v1/*` endpoint — request/response contracts |
| Component Tests | Jest + React Testing Library | UI components — render, interact, assert |
| End-to-End Tests | Playwright | Full browser workflows — register, apply, verify, etc. |
| Performance Tests | Playwright + custom timing | Page load, API response times, concurrent load |
| Security Tests | Custom + OWASP ZAP (optional) | SQL injection, XSS, auth bypass, rate limit |
| Smoke Tests | In-app test runner | Quick production health checks from the admin UI |

**Already in package.json:** jest, playwright, @testing-library/react, supertest

## Development Process Integration

### The "Feature + Test" Contract

Every feature PR must include:

```
Feature PR Checklist:
□ Feature code (routes, components, etc.)
□ Unit tests for new validators/utilities
□ Integration tests for new/modified API endpoints
□ Component tests for new/modified UI components
□ E2E test for the user-facing workflow (if applicable)
□ All existing tests pass
□ No TypeScript errors
```

### When Tests Are Written

| Phase | Test Types Written |
|-------|-------------------|
| During backend development | Unit tests + Integration tests |
| During frontend development | Component tests |
| After feature integration | E2E tests |
| Before phase gate | Performance tests + Security tests |
| In production | Smoke tests (via admin UI) |

### Automated Test Execution

```bash
# During development — run on file save (watch mode)
npm run test:watch

# Before commit — full suite
npm run test

# CI/CD pipeline — full suite + coverage
npm run test:ci

# E2E tests — requires running app
npm run test:e2e

# Performance tests — requires running app + seeded data
npm run test:perf

# Security tests — requires running app
npm run test:security

# Smoke tests — runs against live environment
npm run test:smoke
```

### npm Scripts (package.json additions)

```json
{
  "scripts": {
    "test": "jest --runInBand",
    "test:watch": "jest --watch",
    "test:ci": "jest --runInBand --coverage --forceExit",
    "test:unit": "jest tests/unit",
    "test:integration": "jest tests/integration --runInBand",
    "test:component": "jest tests/component",
    "test:e2e": "playwright test tests/e2e",
    "test:e2e:ui": "playwright test tests/e2e --ui",
    "test:perf": "playwright test tests/performance",
    "test:security": "jest tests/security --runInBand",
    "test:smoke": "jest tests/smoke --runInBand",
    "test:coverage": "jest --coverage --coverageReporters=text --coverageReporters=lcov"
  }
}
```

## Test Database Strategy

```
Test DB: hirestream_test (separate from dev/prod)

Before suite:  Create DB + run migrations
Before each:   Truncate all tables (fast, keeps schema)
After suite:   Drop DB

Environment:   TEST_DATABASE_URL=postgresql://hirestream:hirestream@localhost:5432/hirestream_test
```

**Seed data per test file:** Each integration test file seeds only the data it needs. No shared mutable state between test files.

**Isolation:** Tests run with `--runInBand` (sequential) for integration tests to avoid DB race conditions. Unit and component tests run in parallel.
