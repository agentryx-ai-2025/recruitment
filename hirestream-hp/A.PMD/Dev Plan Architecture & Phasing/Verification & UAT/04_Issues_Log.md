# HireStream — Issues Log

**Build reference:** HireStream v1.4.0
**Compliance matrix:** [01_FRS_Compliance_Matrix.md](01_FRS_Compliance_Matrix.md)

## Purpose

Defects, questions, and concerns discovered during verification and UAT. Each issue is linked back to the relevant compliance item ID.

---

## Status Values

| Status | Meaning |
|--------|---------|
| 🔴 Open | Raised, awaiting triage |
| 🟡 In Progress | Being worked on |
| 🔵 Needs Info | Waiting for reviewer clarification |
| 🟢 Resolved | Fixed; awaiting re-verification |
| ⚪ Closed | Verified resolved by original reporter |
| ⛔ Won't Fix | Decision documented; not to be addressed |

---

## Format

| ID | Date | Item Ref | Severity | Reported by | Description | Status | Resolution |
|----|------|----------|:--------:|-------------|-------------|:------:|------------|

**Severity:**
- 🔴 **Blocker** — Prevents use of the feature
- 🟠 **Major** — Significantly degrades the feature
- 🟡 **Minor** — Small issue, workaround exists
- 🟢 **Trivial** — Cosmetic or polish only

## Issue Template (copy-paste for new entries)

```
| HS-001 | 2026-04-14 | 1.18 | 🟠 Major | HTIS / {name} | One-click apply button stays disabled after CAPTCHA check on Safari 16 | 🔴 Open | — |
```

**Fields:**
- **ID:** `HS-###` sequential
- **Date:** YYYY-MM-DD when first raised
- **Item Ref:** ID from the compliance matrix (e.g., `1.18` or `13.4`)
- **Severity:** 🔴/🟠/🟡/🟢
- **Reported by:** Name + organization
- **Description:** What happened, how to reproduce, expected vs actual
- **Status:** Open / In Progress / Needs Info / Resolved / Closed / Won't Fix
- **Resolution:** What was done or the decision reached
