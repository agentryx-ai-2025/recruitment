# HireStream — v1.5.4 Release Notes

**Release date:** 2026-04-15
**Theme:** *HPSEDC Admin overhaul — compliance oversight + welfare SLA + audit visibility*

Final role overhaul. Admin was 87% FRS-complete but was missing the day-to-day HPSEDC oversight tools. This release adds 4 new admin tabs focused on what the regulator actually does: enforce policy, track welfare, audit activity, and manage users.

## What's new

### New tab: Compliance Oversight (MEA policy enforcement)
HPSEDC's primary daily tool. Shows:
- **Coverage tiles (7)** — % of candidate pool with passport / ECR / PCC / Medical / IELTS / PDO / PBBY on file, with target thresholds and colour-coded status
- **Risk flags (4)** —
  - Placed candidates without PDO completed
  - Placed candidates without PBBY enrolment (Emigration Act §10 violation)
  - Placed candidates without passport on file
  - Passports expiring within 6 months

Every flag has the candidate name + ID visible, so admin can take action immediately.

### New tab: Welfare SLA Monitor
Lists every placement where a 30/60/90-day welfare check-in is overdue. Calculated from `placements.startDate` against the configurable `interview.reminder_lead_hours` + `grievance.escalation_days` settings. Sorted by days overdue (longest first), so HPSEDC can triage worst cases first.

### New tab: Users
Full user management — list all users with role badges, active status, last-login date. One-click enable/disable. Supports role segregation on top of RBAC already in place.

### New tab: Audit Log Viewer
API-backed frontend for the `audit_log` table that was always populated but never surfaced. Shows actor (username + role), action (verb), resource type + ID, timestamp, and detail JSON. Filterable server-side by action/resource/user.

### Filled-in gaps

- **Agency rejection (FRS 4.4)** — `PATCH /api/v1/admin/oversight/agencies/:id/reject` with reason. Notifies the agency user + writes audit entry.
- **Reports polish** — interactive tables ready to replace the "View Report" toast placeholders (agency, sector reports).

### New API endpoints

All under `/api/v1/admin/oversight` (admin + superadmin only):

| Method | Path | Purpose |
|---|---|---|
| GET | `/compliance` | PBBY/PDO/ECR aggregates + risk flag lists |
| GET | `/welfare-sla` | Overdue 30/60/90 check-ins |
| GET | `/audit-log` | Paginated audit log with filters |
| PATCH | `/agencies/:id/reject` | Agency rejection with reason (FRS 4.4) |
| GET | `/users` | All users with role + last-login |
| PATCH | `/users/:id/toggle-active` | Enable/disable user |

## Why this matters

Before v1.5.4, HPSEDC admin could count placements but couldn't see:
- Which placed candidates are operating without PBBY (illegal under Emigration Act §10)
- Who's overdue on welfare check-ins (MEA obligation)
- What's happening across the system (audit log was invisible)
- Agency rejections (could only revoke post-approval)

All four are now one click away. This is what turns a "functional portal" into a "regulator-ready portal."
