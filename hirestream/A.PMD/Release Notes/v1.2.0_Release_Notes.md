## New API Endpoints (v1.2)

### Super Admin (`/api/v1/superadmin/*`) — requires `superadmin` role
| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/users` | List all users (optional `?role=` filter) |
| GET | `/users?role=admin` | Filter by role |
| POST | `/users` | Create user of any role (incl. admin, superadmin) |
| PATCH | `/users/:id/role` | Change a user's role |
| PATCH | `/users/:id/active` | Enable/disable a user |
| GET | `/stats` | System stats (role distribution, uptime, env, Node version) |
| POST | `/reset` | Wipe test data (requires `confirmation: "RESET_HIRESTREAM"`) |
| POST | `/reseed` | Wipe + reseed demo data (requires `confirmation: "RESEED_HIRESTREAM"`) |

All mutations logged to `audit_log` table with user, IP, body, status.

### Agency Reviews (`/api/v1/agencies/:id/reviews/*`)
| Method | Endpoint | Auth | Purpose |
|--------|----------|------|---------|
| GET | `/agencies/:id/reviews` | Public | List reviews + average rating for agency |
| POST | `/agencies/:id/reviews` | Candidate only | Submit 1-5 rating + optional title/review |
| GET | `/agencies` | Public | List all verified agencies |

### Resume Parser (`/api/v1/resume/*`)
| Method | Endpoint | Purpose |
|--------|----------|---------|
| POST | `/parse` | Heuristic extraction from pasted resume text — returns `name`, `email`, `phone`, `experience` (years), `skills[]`, `skillsByCategory{}`, `degrees[]`, `preferredCountries[]`, plus `confidence` scores |

### Admin CSV Export (`/api/v1/admin/reports/export/:entity.csv`)
| Entity | Path |
|--------|------|
| candidates | `/export/candidates.csv` |
| jobs | `/export/jobs.csv` |
| applications | `/export/applications.csv` |
| agencies | `/export/agencies.csv` |
| employers | `/export/employers.csv` |
| drives | `/export/drives.csv` |
| placements | `/export/placements.csv` |
| grievances | `/export/grievances.csv` |
| users | `/export/users.csv` (passwords stripped) |

Admin & superadmin roles only. Returns `text/csv` with `Content-Disposition: attachment`.

## Database Schema Changes

### New Table: `agency_reviews`
```sql
CREATE TABLE agency_reviews (
  id VARCHAR PRIMARY KEY DEFAULT gen_random_uuid()::text,
  agency_id VARCHAR NOT NULL REFERENCES recruitment_agents(id),
  candidate_user_id VARCHAR NOT NULL REFERENCES users(id),
  rating INTEGER NOT NULL,  -- 1-5
  title TEXT,
  review TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);
```

### Role Field Extension
The `users.role` column now accepts `superadmin` in addition to the existing four. No migration needed — role is a text column. The super admin must be seeded out-of-band:

```typescript
// scripts/seed.ts now creates:
{ username: "demo_superadmin", email: "demo_superadmin@hirestream.dev",
  role: "superadmin", password: "test123" }
```

## Security Hardening (Day 18)

### Closed Audit Items (29/31)
| Item | CWE | Status |
|------|-----|--------|
| H2 Concurrent login | CWE-1018 | ✅ Single-session enforcement in auth.routes.ts |
| M1 CORS | CWE-942 | ✅ Whitelist `https://hirestream.osipl.dev` only |
| M4 TRACE/OPTIONS | CWE-650 | ✅ Middleware in server/index.ts |
| M5 Autocomplete | CWE-327 | ✅ `autoComplete="off"` on all auth inputs |
| L1 CSP | CWE-693 | ✅ Helmet CSP with strict directives |
| L5 HSTS | CWE-523 | ✅ 1yr max-age, includeSubDomains, preload |
| L13 Cache-Control | CWE-525 | ✅ no-store on all /api/* |
| L15 Email disclosure | CWE-200 | ✅ No govt emails in client HTML |
| T3 URL guessing | — | ✅ Password change uses email token |
| T4 Single session | — | ✅ Same as H2 |
| T5 30-min timeout | — | ✅ Session `maxAge: 30*60*1000` |
| T7 Strong password | — | ✅ 8+ chars, U+L+D+special |
| T9 Method blocking | — | ✅ Same as M4 |

### Day 18 Defence-in-Depth Additions
- **D1 Input sanitization** (`sanitize.middleware.ts`) — strips `<script>`, `<iframe>`, `<object>`, `<embed>`, `<link>`, `<style>`, `<meta>`, `<form>`, `on*=` handlers, `javascript:` URLs from all POST/PATCH/PUT bodies + query strings. Skips password/token/OTP/file fields.
- **D2 Magic-byte file validation** — verifies uploaded PDF/JPEG/PNG signatures match declared MIME. Rejects spoofed extensions.
- **D3 Audit log on Super Admin** — every mutation (user CRUD, role change, DB reset) writes to `audit_log`.
- **D4 Sensitive rate limiter** — 5 attempts / 15min on `/auth/send-otp` and `/auth/request-password-reset`.
- **D5 Server signature scrub** — no `X-Powered-By`, no `Server` header.
- **D6 Body size limit** — 1MB cap on JSON + urlencoded.

### Remaining (Deployment-time only)
- **T6** Real CAPTCHA API key swap (reCAPTCHA or hCaptcha) — stub active; 5-min change once key provisioned
- **L11** Nginx `server_tokens off` — 1-line config change at deployment

Header A/A/A toggle now: **16px / 19px / 24px** (was 14/16/18). Largest option is true accessibility size. Preference persisted in `localStorage["hs-font"]`, applied before render to prevent flash.

## Migration from v1.1.0

```bash
# 1. Pull code
git pull origin main

# 2. Install deps (no new packages)
npm install

# 3. Apply schema
psql "$DATABASE_URL" -c "CREATE TABLE IF NOT EXISTS agency_reviews (id VARCHAR PRIMARY KEY DEFAULT gen_random_uuid()::text, agency_id VARCHAR NOT NULL REFERENCES recruitment_agents(id), candidate_user_id VARCHAR NOT NULL REFERENCES users(id), rating INTEGER NOT NULL, title TEXT, review TEXT, created_at TIMESTAMP DEFAULT NOW());"

# 4. Re-seed (adds demo_superadmin, keeps existing data)
npx tsx scripts/seed.ts

# 5. Rebuild + restart
npm run build
pm2 restart hirestream   # or: systemctl restart hirestream
```
