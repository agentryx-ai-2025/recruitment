# HireStream — v1.5.2 Release Notes

**Release date:** 2026-04-15
**Theme:** *Employer role overhaul — feature parity with agent side*

The third role to get the v1.5 treatment. Employer dashboard was previously a thin job-posting surface with no applicant pipeline access. Now it is a full hiring-manager workspace.

## New in v1.5.2

### Employer dashboard navigation
- Added **Offers**, **Reports**, and **Activity** tabs (previously Overview / My Jobs / Active / Activity only).
- "View Applicants" button on every job card now routes to the full applicant pipeline page.
- Recent Jobs rows on Overview are now clickable.

### New: Offers & Placements tab
Dedicated view for every offer extended from this employer's jobs. Per placement card shows:
- Candidate avatar + name + linked job
- Status (offered / accepted / declined / active)
- Destination, salary, start date
- **Appointment letter** block — upload URL to a signed PDF, or download a template (FRS 3.5 ✅ implementation-complete, was previously metadata-only)
- **Welfare check-in history** — all 30 / 60 / 90-day entries visible read-only with dates + candidate's own outreach notes

### New: Employer Reports tab
Same Reports view as the Agent role, scoped by `jobs.employerId`. Shows:
- Active vs total jobs, applicants, placements, conversion rate
- 7-column funnel breakdown
- Average time-to-placement
- Top destination countries
- Monthly applications trend chart

### Shared (reused) functionality — now accessible as employer
- `/agent/jobs/:id` applicant pipeline (pipeline, search, bulk actions, internal notes, ICS, CSV export, structured interview feedback) — fully functional for employer role
- `/agent/candidates/:id` candidate profile — read-only compliance visibility for employer (edits stay agent-only)

### Backend changes
| Method | Path | Change |
|---|---|---|
| PATCH | `/api/v1/agent/placements/:id/welfare` | Now accepts `employer` role in addition to agent |
| PATCH | `/api/v1/agent/candidates/:id/compliance` | Now accepts `employer` role (for read + light edits) |
| GET | `/api/v1/agent/reports` | Scopes by `jobs.employerId` when caller is an employer |
| PATCH | `/api/v1/agent/placements/:id/appointment-letter` | **New** — attach a URL to the appointment letter (FRS 3.5) |
| GET | `/api/v1/agent/placements` | **New** — list placements for the current user, scoped by role |
