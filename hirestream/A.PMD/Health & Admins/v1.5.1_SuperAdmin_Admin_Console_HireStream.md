## Role Hierarchy (Admin Tier)

| Role | Access Level | Notes |
|------|-------------|-------|
| `admin` | Full — every admin feature | HPSEDC authority, manages the entire system |
| `system_admin` | Technical — monitoring + tools, no business data edits | Dev/ops only |
| `agent` | Business — own agency management, no system access | Recruiting agency operators |
| `employer` | Business — own company management, no system access | Direct employers |
| `candidate` | Self-service — own profile and applications | Job seekers |

**Note:** HireStream uses `admin` as the top-level role (equivalent to HP Tourism's `super_admin`). There is no separate `super_admin` because HPSEDC operates as a single authority. If role splitting is needed later, the middleware supports it — just add the role to the `users` table enum.

## 2. Ops Console (`/admin/ops`) — Tab-by-Tab

The Ops Console is the central health and monitoring dashboard. Auto-refreshes at configurable interval (15s / 30s / 1m / Off).

*Full tab specification in `v1.5.0_OpsConsole_Plan_HireStream.md`.*

Summary of 10 tabs:
1. **Overview** — Health score, smart alerts, process status, dependencies, sessions
2. **Resources** — CPU, memory, disk, Node.js runtime
3. **Pipeline** — Placement workflow health, stuck apps, agency verification, drives, grievances
4. **Backups** — Scheduler, history, manual trigger
5. **Signals** — Trend analytics with period filters
6. **Logs** — Structured log viewer with filters
7. **Log Query** — RegExp grep across logs
8. **Lookup** — Entity search across candidates/agencies/jobs/applications
9. **DB Explorer** — Pre-built reports + SQL sandbox
10. **System** — TCP, FDs, processes, restart/flush actions

## 3. Super Console (`/admin/super-console`) — Section-by-Section

Admin only. Full system control panel.

| Section | Capabilities |
|---------|-------------|
| **System Overview** | DB connection status, pool stats, version info, environment |
| **Reset Zone** | Granular data reset (by table/type), full DB wipe (with confirmation), selective seed |
| **Communication Settings** | Email (SMTP) config — host, port, user, from address; test-send button. SMS config — provider, API key; test-send button |
| **Notification Settings** | Portal-wide notification preferences: which events trigger email/SMS/in-app |
| **Security** | Rate limit configuration (per-endpoint overrides), session timeout, password policy |
| **External Integrations** | UIDAI API config (endpoint, API key), HIM Access SSO config (client ID, redirect URI), DigiLocker API config (endpoint, API key) — presence/status only, values masked |
| **Seed Accounts** | Create admin accounts, system_admin accounts, test candidate/agent/employer accounts |
| **Smoke Tests** | Run pre-built end-to-end checks: auth flow, DB connectivity, file upload, email send, SMS send, external API reachability |
| **Test Data** | Create synthetic test candidates, agencies, jobs, applications for staging/demo |
| **Feature Toggles** | Enable/disable: agency self-registration, OTP login, DigiLocker upload, AI matching, bilingual toggle, grievance submission |
| **Placement Rules** | Configure: max applications per candidate, auto-match threshold, interview scheduling rules |
| **Verification Config** | Per-agency-type verification requirements (document checklist, license mandatory, past-record mandatory) |

## 4. User Management (`/admin/users`)

Available to `admin`, `system_admin`.

- **List all users** — searchable, filterable by role, district/location, registration date
- **Active Sessions panel** — who is currently logged in, by role
- **Create user** — any role, any location
- **Edit user** — name, email, phone, role, location
- **Activate / Deactivate** — block a user without deleting their data
- **View user audit trail** — all actions taken by/on this user

### Role assignment capabilities

| Acting Role | Can Create/Edit |
|-------------|----------------|
| `admin` | Any role including other admins, system_admins, agents, employers, candidates |
| `system_admin` | View only (no writes) |

## 5. Settings (`/admin/settings`)

### Available to `admin` + `system_admin` (read/toggle)

| Setting | Description |
|---------|-------------|
| **Test Mode** | Switch between live external APIs and mock/simulated mode |
| **OTP Authentication** | Enable/disable OTP for login (fallback to password-only) |
| **Agency Self-Registration** | Allow agencies to register themselves or require admin creation |
| **Service Toggle** | Enable/disable individual features (job posting, applications, drives, grievances) |
| **Rate Limits** | View current rate limit thresholds per endpoint group |
| **Session Timeout** | View session timeout duration |
| **File Upload Limits** | Max file size, allowed MIME types for document uploads |
| **AI Matching** | Enable/disable AI match score calculation |
| **Bilingual Mode** | Enable/disable Hindi language toggle for candidates |

### `admin` only (write/modify)

| Setting | Description |
|---------|-------------|
| **Rate Limits** (write) | Change per-endpoint rate limit values |
| **Session Timeout** (write) | Change session timeout duration |
| **Max Applications** | Max concurrent applications per candidate |
| **Auto-Match Threshold** | Minimum match score for auto-recommendation |
| **Verification SLA** | Target days for agency verification turnaround |
| **Grievance SLA** | Target days for grievance resolution |
| **Drive Approval Auto-Expire** | Days after which unapproved drive requests expire |
| **Super Console Toggle** | Enable/disable Super Console section visibility |
| **Maintenance Mode** | Put portal in read-only maintenance mode with custom message |

## 6. Backup Management (`/admin/backup`) — `admin` only

- **Run backup now** — triggers `pg_dump --format=custom --compress=9`
- **Backup scheduler** — start/stop cron schedule, set cron expression
- **Backup history** — list with size, timestamp, status
- **Download** — download any backup file (binary or SQL format)
- **Delete** — remove old backup files
- **Scheduler status** — running/stopped, next scheduled run
- **Verify backup** — test-restore a backup to verify integrity (does not affect live DB)

## 7. Reports & Analytics (`/admin/reports`)

Available to `admin`, `system_admin`.

### Placement Reports

| Report | Description |
|--------|-------------|
| **By District** | Candidates registered and placed, grouped by district/location |
| **By Agency** | Applications handled per agency, placement success rate, avg time-to-placement |
| **By Skill** | Top skills in demand (from jobs) vs. available (from candidates), gap analysis |
| **By Placement Status** | Full funnel: registered → applied → shortlisted → interviewed → selected → placed |
| **By Country** | Placements by destination country, salary ranges, popular sectors |
| **By Sector** | Job postings and placements by industry sector |

### Operational Reports

| Report | Description |
|--------|-------------|
| **Agency Performance** | Verification turnaround, placement count, candidate rating, active jobs |
| **Application Throughput** | Applications processed per day/week/month, avg review time |
| **Grievance Summary** | Open/closed/pending, avg resolution time, by category |
| **Drive Effectiveness** | Candidates per drive, selection rate, placement rate |
| **User Activity** | Registrations over time, active users, login frequency by role |

### Export Options
- **CSV** — all reports exportable as CSV
- **PDF** — formatted report with header, date range, charts (via pdf-lib/puppeteer)
- **Scheduled email** — configure daily/weekly report delivery to admin email

## 8. Agency Management (`/admin/agencies`)

Available to `admin`, `system_admin`.

### Agency Verification Queue
- List of agencies pending verification (sorted by registration date)
- For each agency:
  - Company name, license number, registration date
  - Uploaded documents (license, past-record, company registration)
  - Document viewer (inline PDF/image preview)
  - Specializations claimed
  - **Actions:** Approve / Reject / Request More Info
  - Rejection requires reason (recorded in audit log)

### Agency Directory
- All registered agencies with status (verified / pending / rejected / revoked)
- Filter by: status, specialization, location, registration date
- Per-agency detail:
  - Profile information
  - Active job postings
  - Placement history
  - Candidate reviews/rating
  - Verification history
  - **Actions:** Revoke verification, Edit profile, Deactivate

### Agency Analytics
- Top agencies by placements
- Average time from registration to verification
- Agency-candidate satisfaction correlation

## 9. Recruitment Drives (`/admin/drives`)

Available to `admin`, `system_admin`.

- List of all recruitment drives (upcoming, active, completed, cancelled)
- **Pending Approval Queue:**
  - Drive details: agency, date/time, location, target roles, expected candidates
  - **Actions:** Approve / Reject / Request Changes
- **Drive Calendar View** — visual calendar of all scheduled drives
- **Drive Outcomes:**
  - Post-drive reports: candidates attended, interviewed, selected, placed
  - Drive effectiveness metrics

## 10. Grievance Management (`/admin/grievances`)

Available to `admin`, `system_admin`.

- **Grievance Queue** — all submitted grievances, filterable by status/category/date
- **Categories:** Agency complaint, Application issue, Technical problem, Policy inquiry, Other
- **Workflow:** Submitted → Under Review → Action Taken → Resolved / Escalated
- **Per-grievance:**
  - Submitter details (candidate/agency)
  - Grievance text + attachments
  - Admin notes (internal)
  - Resolution notes (visible to submitter)
  - **Actions:** Assign to self, Add note, Resolve, Escalate, Close
- **SLA Tracking:** Days open, time to first response, time to resolution
- **Grievance Analytics:** Volume trends, common categories, avg resolution time

## 11. FAQ & Announcements (`/admin/faq`)

Available to `admin`.

### FAQ Management
- CRUD for FAQ entries (question + answer, categorized)
- Categories: Registration, Job Application, Agencies, Overseas Placement, Technical Support
- Ordering (drag-and-drop or manual sort order)
- Bilingual support: English + Hindi for each entry

### Portal Announcements
- Create time-bound announcements (title, body, start date, end date)
- Target audience: All users / Candidates only / Agencies only / Employers only
- Pin/unpin announcements
- Announcement displayed as banner on dashboard

## 13. Audit Log (`/admin/audit-log`)

Available to `admin`, `system_admin`.

- **Every admin action** is logged:
  - Who (user ID, role, IP address)
  - What (action type: create, update, delete, verify, reject, login, logout)
  - Where (resource type: user, agency, job, application, drive, grievance, setting)
  - When (timestamp)
  - Details (before/after values for updates)
- **Filters:** by user, by action type, by resource type, by date range
- **Export:** CSV export for compliance documentation
- **Retention:** 1 year minimum (ISO 27001 requirement)
