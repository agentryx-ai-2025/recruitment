## 8 · CARD — How it's built (1:30)

**[SCREEN]** The 3-tier architecture diagram (from the Tech Stack PDF): Presentation (React SPA · React Native apps) → Application (Node/Express · RBAC · matching engine · audit) → Data (PostgreSQL). Side panel: Security · Standards. Logos/labels kept light.

**[VO]**
> A word on how HireStream is built — because for a government platform, the engineering matters as much as the features.
>
> It follows a clean three-tier architecture. The presentation tier is a responsive web application and native Android and i O S apps. The application tier runs on Node-dot-J S, with role-based access control, the candidate-matching engine, and a security layer of hardened sessions, rate limiting, and input validation. And the data tier is PostgreSQL, with roughly thirty-six tables and an immutable audit log.
>
> Every technology is modern, proven, and open-source — no licensing lock-in. Security is built in: role-based access control for each of the four roles, encryption in transit and at rest, bcrypt-hashed passwords, one-time passwords, and a complete audit trail. And it is built to standards — G I G W accessibility, I S O twenty-seven-thousand-one-aligned security practices, and data-protection equivalent to G D P R.
>
> The external integrations H P S E D C requires — HIM Access single sign-on, Aadhaar, DigiLocker, and the government email and S M S gateways — are all built as pluggable adapters, ready to switch on from the admin console the moment production credentials are provided. No code change required.

**[VO]**
> To take HireStream into production, we have one ask of the I T department — and we have made it precise. We request two virtual machines on the State Data Centre: a staging server and a production server, sized for the five-thousand-concurrent-user target in the F R S. We have specified the exact base software to pre-install — Ubuntu, Node, PostgreSQL, and Nginx — because we propose an air-gapped model, with the servers isolated from the internet and accessed only over S S H through your jump host. H T I S delivers the application as a pre-built artifact; no build ever runs on the server.
>
> Beyond that, we need C A-issued T L S certificates for the two domains, controlled outbound reachability to the government services the integrations call, and a standard service account and backup target. The full, itemised request is in the Infrastructure Provisioning document — designed to slot straight into your existing data-centre policies, and to make it easy to say yes.
